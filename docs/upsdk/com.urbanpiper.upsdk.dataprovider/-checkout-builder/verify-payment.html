verifyPayment

fun verifyPayment(transactionId: String, gwTxnId: String, transactionStatus: Int, callback: Callback<PaymentCallbackResponse>): CancellableTask

This step is only required if the payment did not happen through a redirection flow (i.e - through a webview with a redirection url from the payment init response) This Marks the completion of a transaction.

Parameters

transactionId -

  • Transaction id from payment init

gwTxnId -

  • payment gateway transaction id

transactionStatus -

  • transaction status, it can have the following values 0 - Transaction success 1 - Transaction failed 5 - Transaction cancelled

callback -

  • Callback to return the result

Return
CancellableTask - the request can be cancelled by calling .cancel() on the CancellableTask
